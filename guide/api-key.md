# 创建 API Key

API Key 是客户端访问 Recurdream 的凭证。请把它当作密码一样保管。

## 创建步骤

1. 登录 `https://api.recurdream.com`
2. 打开 **API 密钥**
3. 点击 **创建密钥**
4. 选择分组或模型权限
5. 按需设置额度、有效期、IP 限制
6. 创建后复制 API Key

## 建议配置

| 配置项 | 建议 |
| --- | --- |
| 名称 | 按客户端或用途命名，例如 `tavo-main` |
| 分组 | 选择你要使用的模型分组 |
| 额度金额 | 个人使用可先留空；团队或测试 Key 建议设置上限 |
| 有效期 | 长期使用可永久有效；临时测试建议设置过期时间 |
| IP 限制 | 固定服务器调用时建议设置白名单 |

## Key 格式

Recurdream 的 API Key 会在客户端里作为 Bearer Token 使用：

```text
Authorization: Bearer YOUR_API_KEY
```

大多数客户端只需要你把 Key 粘贴到 `API Key` 输入框，不需要手动写 `Bearer`。

## 安全提醒

- 不要把 API Key 发给陌生人。
- 不要把 API Key 写入公开 GitHub 仓库。
- 不同客户端建议使用不同 Key，方便单独停用和查用量。
- Key 泄露后请立即在后台禁用或删除，并创建新 Key。

## 常见问题

### 创建 Key 后不能用

优先检查：

- Key 是否分配了分组。
- 所选分组是否有可用模型和账号。
- 账户余额是否充足。
- 客户端 Base URL 是否填写正确。

### 一个 Key 可以给多个客户端用吗

可以，但不推荐。分开创建 Key 更方便统计用量，也方便发现异常时单独停用。
